1. Introduction — what changed

The victim is no longer an open probe. Adding R2 and C2 turns the secondary into its own series L₂–R2–C2 loop, driven by the coupled EMF M dI₁/dt. What we now measure is the voltage across C2 — the induced EMF after an R2C2 low-pass filter. Two consequences: (1) the victim waveform stops looking like dI₁/dt and instead tracks the integral of the secondary current, i.e. it follows I₁(t); (2) the peak is heavily attenuated — here a 500 mV EMF spike becomes a 100 mV bump (≈5× smaller) — a real, measurable victim. Everything below is first-principles: coupled KVL, a closed-form estimate, and a full 4-state simulation that confirms it.

2. Completed schematic & loop equations

Figure 1 — Primary series-RLC (I₁) coupled by M to a secondary L₂–R2–C2 loop (I₂); victim = voltage across C2.
primary loop
$$V_{agg}=R_1 I_1+L_1\dot I_1+V_{C_1}+M\dot I_2$$
secondary loop
$$0=M\dot I_1+L_2\dot I_2+R_2I_2+V_{C_2}$$
victim peak (slow-filter limit)
$$V_{victim,pk}\approx\frac{M}{R_2C_2}\,I_{1,pk}$$

🔄 Closed-form vs sim — when they diverge. The gate compares the coupled RC filter time τ2=RinCin to the aggressor pulse width (≈100 ps here):
  • Slow / heavy filter (τ2 >> pulse, e.g. Rin=2 kΩ, Cin=5 pF → τ2=10 ns): the closed form is the exact limit — sim and closed form agree (≈0.095 mV vs 0.099 mV).
  • Fast / light filter (τ2 ≲ pulse, e.g. Rin=50 Ω, Cin=0.1 pF → τ2=5 ps): Cin partly discharges through Rin during the pulse, so the true peak is far lower — the closed form over-reads (can be 2–11×). Here the (sim) column is the trustworthy number; the closed form stays as a safe upper bound.
(A big closed-form–sim gap therefore flags a fast victim filter, not an error.)
Modeling assumptions (first approach). (1) Worst-case voltage step for the aggressor edge (dV/dt=∞ limit; inductance bounds dI/dt) — finite dV/dt only lowers the peak, so dV/dt & f are captured but not yet de-rated. (2) I1,pk≈Vagg/Rout sets the loop current. (3) Coupling length = trace overlap; coplanar filament M(d). (4) The closed form is the slow-filter limit — it over-reads when τ2=RinCin approaches the pulse width (e.g. Case B: 177 mV vs 100 mV exact), so it is a conservative spec gate; peak also capped at Vagg. (5) Only the inductive path is scored — capacitive / ground-return / PDN are listed but not yet computed. Verify against the full 4-state sim (§8–§10) or a 3-D field solver before tape-out.

8. First-principles derivation

Coupled KVL with the mutual inductance matrix (states \(I_1,V_{C_1},I_2,V_{C_2}\)):

$$\begin{bmatrix}L_1&M\\M&L_2\end{bmatrix}\!\begin{bmatrix}\dot I_1\\\dot I_2\end{bmatrix} =\begin{bmatrix}V_{agg}-R_1 I_1-V_{C_1}\\-R_2I_2-V_{C_2}\end{bmatrix},\qquad \dot V_{C_1}=\frac{I_1}{C_1},\quad \dot V_{C_2}=\frac{I_2}{C_2}$$

Secondary is R-dominated for this fast pulse (\(\omega L_2\ll R_2\)), so the induced EMF drops mostly across \(R_2\): \(\;I_2\approx -\dfrac{M}{R_2}\dot I_1\). Feeding that into the cap and integrating:

$$V_{victim}=\frac{1}{C_2}\!\int I_2\,dt\approx-\frac{M}{R_2C_2}\!\int \dot I_1\,dt =-\frac{M}{R_2C_2}\,I_1(t)$$ $$\boxed{\,V_{victim,pk}\approx\frac{M}{R_2C_2}\,I_{1,pk} =\frac{M\,V_{agg}}{R_1\,R_2\,C_2}\quad(\text{since }I_{1,pk}\approx V_{agg}/R_1)\,}$$

That boxed formula is the slow-filter limit (τ2 >> pulse). Two caveats: (1) the primary R1 must appear — it sets I1,pk=Vagg/R1; your sketch's box "2M Vagg/R2C2" dropped it (and used a rectangular area, hence the factor 2). (2) With the light filter chosen here (τ2=50 ps ≈ the ~100 ps pulse) we are not in the slow-filter limit, so that formula over-reads — only the full 4-state solve is exact.

10. Python model (coupled 4-state)

Self-contained — copy, paste, run with nothing but the Python standard library (no numpy / scipy / matplotlib). It builds the mutual inductance from the geometry, RK4-solves the 4-state ODE, and prints an ASCII waveform with a Verr spec line — the trace beyond spec is red, the part within spec green, with the peak marked (*) and labeled — reproducing the default-populated Spec Checker case (§5–7): aggressor PWM_HDRV into ISEN_CS1 (Rin=200 Ω → 3.18 mV, FAIL) and ISEN_CS2 (Rin=1000 Ω → 0.85 mV, PASS vs the 1 mV budget). Same file in the repo: inductive_crosstalk_model.py.

# Coupled 4-state inductive-crosstalk model — copy, paste, run (pure standard library).
#   python3 this_file.py          # no numpy / scipy / matplotlib needed; prints an ASCII waveform
#
# Reproduces the DEFAULT case populated in the Spec Checker (section 7):
#   aggressor PWM_HDRV : dV=5 V, Rout=100 ohm, 30x10 um trace, 4000 um long
#   victims  ISEN_CS1 (Rin=200 ohm) and ISEN_CS2 (Rin=1000 ohm), Cin=1 pF, d=100 um, 150 um long
# The waveform is drawn with a Verr spec line; the part of the trace BEYOND spec is red, the
# part within spec is green (ANSI colors — most terminals show them).
import math
mu0 = 4e-7*math.pi
RED, GREEN, RESET = "\033[91m", "\033[92m", "\033[0m"

def Lself(L,w,t): return mu0*L/(2*math.pi)*(math.log(2*L/(w+t))+0.5+0.2235*(w+t)/L)  # trace self-L
def Mmut(d,L):    return mu0*L/(2*math.pi)*(math.asinh(L/d)-math.sqrt(1+(d/L)**2)+d/L) # parallel mutual

# --- default-populated geometry / drive ---
Vagg, R1, C1 = 5.0, 100.0, 100e-12
La = Lself(4000e-6, 30e-6, 10e-6)      # aggressor self-inductance (4 mm trace)
Lv = Lself( 150e-6, 30e-6, 10e-6)      # victim self-inductance (150 um trace)
M  = Mmut(100e-6, min(4000e-6, 150e-6))# mutual over the 150 um overlap at d = 100 um

def simulate(R2, C2):
    """RK4 solve of x=[I1,Vc1,I2,Vc2]; returns (times, victim voltage across C2)."""
    det = La*Lv - M*M
    def f(x):
        I1,Vc1,I2,Vc2 = x
        b1 = Vagg - R1*I1 - Vc1                 # primary branch
        b2 =       - R2*I2 - Vc2                # secondary branch (no source)
        return (( Lv*b1 - M*b2)/det, I1/C1, (-M*b1 + La*b2)/det, I2/C2)
    dt = min(La/R1, Lv/R2, R2*C2, math.sqrt(Lv*C2))/25
    T  = max(8*R2*C2, 40*La/R1, 3e-9); N = min(60000, math.ceil(T/dt)); dt = T/N
    x=[0.0]*4; ts=[0.0]; vs=[0.0]
    for i in range(N):
        k1=f(x); x2=[x[j]+.5*dt*k1[j] for j in range(4)]
        k2=f(x2); x3=[x[j]+.5*dt*k2[j] for j in range(4)]
        k3=f(x3); x4=[x[j]+dt*k3[j] for j in range(4)]; k4=f(x4)
        for j in range(4): x[j]+=dt/6*(k1[j]+2*k2[j]+2*k3[j]+k4[j])
        ts.append((i+1)*dt); vs.append(x[3])
    return ts, vs

def _paint(chars, cols):
    """chars: list of characters; cols: list of None/'r'/'g' -> ANSI string (run-length grouped)."""
    out=""; i=0; n=len(chars)
    while i<n:
        c=cols[i]
        if c is None: out+=chars[i]; i+=1
        else:
            j=i
            while j<n and cols[j]==c: j+=1
            out+=(RED if c=='r' else GREEN)+"".join(chars[i:j])+RESET; i=j
    return out

def ascii_plot(ts, vs, title, Verr, W=60, H=17):
    """ASCII waveform with a Verr spec line; trace beyond +/-Verr is red, within is green."""
    ip = max(range(len(vs)), key=lambda i: abs(vs[i])); vpk, tpk = vs[ip], ts[ip]
    Tplot = min(max(9*tpk, 300e-12), ts[-1])
    lo = min(0.0, min(vs), -Verr*1.15); hi = max(0.0, max(vs), Verr*1.15)  # keep +/-Verr in view
    row = lambda v: min(H-1, max(0, int(round((hi-v)/(hi-lo)*(H-1)))))
    col = lambda t: min(W-1, max(0, int(round(t/Tplot*(W-1)))))
    grid=[[' ']*W for _ in range(H)]; cols=[[None]*W for _ in range(H)]
    rSp, rSm, r0 = row(Verr), row(-Verr), row(0.0)
    for c in range(W):                                   # dashed spec lines at +/-Verr
        if c%3==0:
            if grid[rSp][c]==' ': grid[rSp][c]='.'
            if grid[rSm][c]==' ': grid[rSm][c]='.'
    for c in range(W): grid[r0][c]='-'                   # zero axis
    pkc=col(tpk); j=0
    for c in range(W):                                   # curve (on top), colored by spec
        tt=c/(W-1)*Tplot
        while j+1<len(ts) and ts[j+1]<tt: j+=1
        r=row(vs[j]); beyond = abs(vs[j])>Verr
        grid[r][c] = '*' if c==pkc else 'o'
        cols[r][c] = 'r' if beyond else 'g'
    ann=' <- peak = %.3f mV'%(vpk*1e3); rp=row(vpk); s=max(0,min(pkc+2,W-len(ann)-1))
    for i,ch in enumerate(ann):                          # peak label (color = peak's spec state)
        if s+i<W and grid[rp][s+i]==' ':
            grid[rp][s+i]=ch; cols[rp][s+i] = 'r' if abs(vpk)>Verr else 'g'
    slab=' spec (Verr)'                                  # spec-line tag on the -Verr row
    s2=W-len(slab)
    for i,ch in enumerate(slab):
        if 0<=s2+i<W and grid[rSm][s2+i] in ' .': grid[rSm][s2+i]=ch
    ylab={0:'%7.3f'%(hi*1e3), r0:'%7.3f'%0.0, rSm:'%7.3f'%(-Verr*1e3), H-1:'%7.3f'%(lo*1e3)}
    out=[title]
    for ri in range(H):
        out.append(ylab.get(ri,'       ')+' |'+_paint(grid[ri], cols[ri]))
    out.append('        +'+'-'*W)
    out.append('        0'+' '*(W//2-4)+'time [ps]'+' '*(W//2-13)+'%d'%round(Tplot*1e12))
    return '\n'.join(out), vpk

print("La=%.3f nH   Lv=%.4f nH   M=%.2f pH   (Vagg=%.0f V, R1=%.0f ohm, Cin=1 pF)\n"
      % (La*1e9, Lv*1e9, M*1e12, Vagg, R1))
Verr = 1e-3
for name, Rin in [("ISEN_CS1", 200.0), ("ISEN_CS2", 1000.0)]:
    ts, vs = simulate(Rin, 1e-12)
    plot, vpk = ascii_plot(ts, vs, "PWM_HDRV -> %s  (Rin=%.0f ohm)  [victim V across C2, mV]"%(name,Rin), Verr)
    ok = abs(vpk) <= Verr
    print(plot)
    print("        Verr = %.3f mV   ->   %s%s%s\n" % (Verr*1e3, GREEN if ok else RED, "PASS" if ok else "FAIL", RESET))

12. Design question asked in an interview

Q1 [SI role — ClimbTheLadder[4]]
“A victim net has a receiver R and C to ground. Does that make crosstalk better or worse, and why?”
Answer: The R2C2 forms a low-pass that integrates the induced EMF (M dI₁/dt), so the measured victim tracks I₁(t) and the peak drops by ≈τ2/(pulse width) — here 500 mV → 100 mV with a light filter. A slower (heavy) R2C2 attenuates far more, and R1 caps the achievable peak [1][2].
Q2 [HW/SI role — Interviews.chat[5]]
“What causes ground bounce / SSN, and how do you reduce it?”
Answer: V=L dI/dt across shared return inductance. Cut L, dI/dt, or M — and, as §4 shows, a slower victim RC further suppresses the bump: Vvictim,pk≈MVagg/(R1·R2C2) [1].
Q3 [SI role — ClimbTheLadder[4]]
“Why is a faster edge worse even if clock frequency is unchanged?”
Answer: Coupled noise scales with slew (dI/dt). A faster edge raises the EMF spike; only the part within the victim filter’s band reaches C2, but the integral (charge) still grows with M [2].
